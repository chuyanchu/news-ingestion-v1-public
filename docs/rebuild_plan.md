# 财经新闻板块热度模块重构方案

## 目标

把原本偏“新闻采集平台”的模块收敛成消息驱动的板块热度看板：

- 每日热力图：展示各股票板块热度。
- 点击板块：查看相关新闻和关键催化。
- 实时消息推送：定时刷新新消息，命中板块后进入看板。
- 交易池：按过去 1/3/5/10 天筛选持续有消息催化的板块。
- GPT 增强：可选生成摘要、影响方向、重要性评分。

## 设计原则

- 保留现有抓取方法，不推倒重来。
- 第一版不用 BERTopic 等重模型，先用板块词典 + 规则评分保证可跑通。
- GPT 是增强项，不配置 key 时系统仍可演示。
- 前端只做一个清晰看板，不继续堆复杂导出和质量报告入口。

## MVP 接口

```text
GET  /
GET  /api/v1/realtime?date=YYYYMMDD&limit=20
GET  /api/v1/heatmap?date=YYYYMMDD&limit=30
GET  /api/v1/sector/{sector_name}/news?date=YYYYMMDD
GET  /api/v1/trade-pool?days=3&limit=20
GET  /api/v1/gpt/status
POST /api/v1/refresh
```

## 评分口径

板块热度 = 新闻热度 + 来源权重 + 时效性 + 重要词加权 + 相关新闻数量加权。

重要词包括：政策、重磅、突破、订单、中标、涨价、并购、重组、业绩、投产、获批等。

## 回测计划

第一阶段只保存每日板块热度结果；第二阶段接行情数据后验证：

- 热度 Top N 板块次日表现。
- 连续进入交易池板块的 3 日/5 日收益。
- GPT 重要性评分与后续涨幅的相关性。
